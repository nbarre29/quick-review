## 1Ô∏è‚É£ `Hashtable` is a legacy class (pre-Java 1.2)

* Introduced in **Java 1.0**
* Designed **before**:

  * Collections Framework
  * Modern concurrency
  * Generics

`HashMap` was introduced later as part of the **Collections Framework**.

## 2Ô∏è‚É£ Synchronization model is the BIGGEST problem

### Hashtable (synchronized everywhere ‚ùå)

```java

public synchronized V put(K key, V value) { ... }
```

* Every method locks the **entire table**
* Only **one thread** can read/write at a time
* Massive performance bottleneck

---

### HashMap (not synchronized ‚úÖ)

```java

```

* No locking
* Very fast
* Perfect for **single-threaded or externally synchronized use**

---

### Modern alternative ‚Üí `ConcurrentHashMap` ‚≠ê

```java

```

* Fine-grained locking
* Lock-striping / CAS
* High concurrency
* Used heavily in real systems

## 4Ô∏è‚É£ Poor scalability

### Hashtable

* One global lock
* Threads block each other
* Does not scale on multi-core systems

### ConcurrentHashMap

* Bucket-level locking
* Reads are non-blocking
* Scales very well
